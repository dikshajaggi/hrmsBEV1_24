🔄 REFRESH TOKEN FLOW (PLAIN ENGLISH)
1️⃣ User logs in
2️⃣ Backend gives:

access token (15 min)
refresh token (30 days, cookie)

3️⃣ Access token expires
4️⃣ Frontend calls /auth/refresh
5️⃣ Backend:

checks refresh token
issues new access token
rotates refresh token (important)
User never notices anything.


🔐 IMPORTANT SECURITY RULE (SENIOR LEVEL)
Refresh tokens are ONE-TIME USE

Every refresh:
old refresh token → invalid
new refresh token → issued
This prevents token theft attacks.

🧠 What “logout” REALLY means (simple words)

Logout does NOT mean:
deleting access token (you can’t)
trusting frontend to “forget” it
Logout ACTUALLY means:
kill the refresh token
so no new access token can ever be issued

After that:
access token dies naturally in 15 minutes
user is effectively logged out everywhere


🔄 LOGOUT FLOW (PLAIN ENGLISH)
1️⃣ User clicks logout
2️⃣ Frontend calls /auth/logout
3️⃣ Backend:
reads refresh token from cookie
finds it in DB
marks it as revoked
clears cookie
4️⃣ Done ✅

Audit log = “who did what, on which record, and when”

❓ Why do we need audit logs in an HRMS?
Because HRMS is a sensitive system.


Without audit logs:
Manager says: “I never approved this leave”
HR says: “System auto-approved”
Employee says: “Someone changed my data”
👉 No proof. No accountability. Big problem.

With audit logs:
You can prove exactly what happened

🔍 What actions should be audited?
Not everything. Only important actions.

✅ MUST audit
HR approves / rejects user
Manager approves / rejects leave
HR overrides leave
Attendance override
Role changes
User disable / enable

❌ No need to audit
Viewing data
Login success
Fetching lists

🧱 How audit logs look in DB (simple)
You already had this table in your schema 👇

AuditLog
--------------------------------
id
entity        (LEAVE, USER, ATTENDANCE)
entityId      (which record)
action        (APPROVED, REJECTED, OVERRIDDEN)
performedBy   (who did it)
createdAt


That’s it. No magic.

🛠 How audit logs are used (simple flow)
Example: Manager approves leave

When manager approves:
Update leave status
Insert audit log
Both happen together.


We will add audit logs in transactions, so data + log are always in sync.
